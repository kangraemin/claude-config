# Library Evolve

Library의 **내용**이 아니라 **운영 규칙과 구조 자체**를 개선한다.
Cog 프로젝트의 /evolve 패턴에서 영감.

## 분석 항목

### 1. 카테고리 밸런스
- LIBRARY.md에서 각 카테고리별 항목 수 집계
- 한 주제에 10개+ 파일이 몰려 있으면 → 서브카테고리 분리 제안
- 1개만 있는 주제가 여러 개면 → 상위 주제로 병합 제안

### 2. 템플릿 사용률
- 최근 10개 지식 파일의 type 분포 확인
- 특정 유형만 쏠리면 → 다른 유형 저장 누락 가능성 알림
- 템플릿이 안 맞는 패턴 발견 시 → 새 템플릿 제안

### 3. TAXONOMY.md 정합성
- 실제 폴더 구조와 TAXONOMY.md의 불일치 발견
- 쓰이지 않는 카테고리 정리 제안
- 새로 필요한 카테고리 제안

### 4. 규칙 효과성
- GUIDE.md 규칙 중 실제로 안 지켜지는 것 발견
  - 예: durability 태그가 빠진 파일이 많으면 → 규칙이 너무 무거운 건 아닌지
- Prediction Error 필터가 제대로 작동하는지 (당연한 내용이 저장되고 있진 않은지)

### 5. Synthesis 갭
- 3개+ 주제가 관련되어 있는데 synthesis가 없는 클러스터 발견
- synthesis 후보 제안

### 6. 검색 효과성
- library_search가 자주 빈 결과를 반환하는 패턴이 있는지
- 파일명/내용의 키워드가 검색 쿼리와 매칭 안 되는 경우 → 키워드 보강 제안

## 플로우

1. **수집**: LIBRARY.md, TAXONOMY.md, GUIDE.md, 최근 지식 파일 10-20개 읽기
2. **분석**: 위 6개 항목 체크
3. **제안**: 개선 사항 목록 출력 (자동 적용 안 함)
4. **사용자 확인**: 적용할 항목 선택
5. **실행**: 선택된 항목만 적용
6. **CHANGELOG**: 변경 사항 기록
7. **커밋**: git commit & push

## 출력 형식

```
🔬 Library Evolve 분석 결과

📊 현황:
- 전체: N개 주제, M개 파일
- 카테고리 분포: [막대 그래프 or 표]

💡 개선 제안:
1. [높음] 설명 — 근거
2. [중간] 설명 — 근거
3. [낮음] 설명 — 근거

적용할 항목 번호를 선택하세요 (예: 1,3)
```
