# /handoff

세션을 빠르게 마무리한다. 다음 세션이 이어받을 수 있도록 HANDOFF.md를 남기고 끝.

---

## 실행 흐름

### Step 1: HANDOFF.md 작성

프로젝트 루트에 `HANDOFF.md`를 생성(또는 덮어쓰기)한다.

대화 컨텍스트에서 다음을 추출:

```markdown
# Handoff

## 이번 세션에서 한 일
- 완료한 작업 (간결하게)

## 미완료 / 다음에 할 일
- 남은 작업과 구체적 다음 액션
- 시도했지만 실패한 것이 있으면 원인 포함

## 참고
- 다음 세션에서 알아야 할 컨텍스트 (파일 경로, 결정사항 등)
```

**작성 기준:**
- 다음 세션의 Claude가 이 파일만 읽고 이어갈 수 있을 정도로 구체적으로.
- 하지만 장황하지 않게. 불릿 포인트 위주.
- 미완료 작업이 전혀 없으면 "미완료 / 다음에 할 일" 섹션에 "없음"으로 표기.

### Step 2: Memory 갱신

auto memory `MEMORY.md`를 확인하고, 이번 세션에서 확정된 사실 중 기록할 만한 게 있으면 갱신한다.
- 기존 내용과 중복이면 업데이트 (새로 추가하지 않음)
- 200줄 이내 유지
- 확정되지 않은 정보는 넣지 않음
- 갱신할 게 없으면 스킵

### Step 3: 커밋

미커밋 변경사항이 있으면 `/commit` 스킬 실행.
없으면 스킵.

### Step 4: 안내

```
✅ 핸드오프 완료. `/clear`로 컨텍스트를 정리하세요.
```
